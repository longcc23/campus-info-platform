# 📝 创建测试账号指南

## 🔐 账号密码设置

管理后台使用 **Supabase Auth** 进行认证，所有用户账号都需要在 Supabase Dashboard 中创建。

---

## ✅ 方法一：通过 Supabase Dashboard 创建（推荐）

### 步骤 1: 访问 Supabase Dashboard

1. 打开浏览器，访问：https://app.supabase.com
2. 登录你的 Supabase 账号
3. 选择项目（项目 URL 中包含 `civlywqsdzzrvsutlrxx`）

### 步骤 2: 创建用户

1. 在左侧菜单中找到 **Authentication**
2. 点击 **Authentication** → **Users**
3. 点击右上角的 **Add user** 按钮
4. 选择 **Create new user**

### 步骤 3: 填写用户信息

在创建用户的表单中填写：

- **Email**: `admin@test.com`（或你想要的邮箱）
- **Password**: `test123456`（或你想要的密码，建议至少 6 位）
- **Auto Confirm User**: ✅ **必须勾选**（这样用户无需邮箱验证即可登录）

### 步骤 4: 确认创建

点击 **Create user** 按钮完成创建。

## ⚠️ 常见问题

### 问题 1: 登录时显示 "Invalid login credentials"

**可能原因**：
- 用户账号未创建
- 密码错误
- 用户未确认（Auto Confirm 未勾选）

**解决方案**：
1. 检查 Supabase Dashboard 中是否已创建用户
2. 确认密码输入正确
3. 确认创建用户时勾选了 "Auto Confirm User"

### 问题 2: 登录后立即被登出

**可能原因**：
- Supabase 配置问题
- Cookie 设置问题

**解决方案**：
1. 清除浏览器 Cookie
2. 检查 Supabase 项目设置中的认证配置

## 🔒 安全建议

1. **开发环境**：可以使用简单的测试密码
2. **生产环境**：必须使用强密码（至少 12 位，包含大小写字母、数字、特殊字符）
3. **账号管理**：定期检查和清理不需要的测试账号
4. **密码管理**：不要在代码或文档中硬编码密码

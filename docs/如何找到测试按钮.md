# 🔍 如何找到测试按钮

## 📍 测试按钮位置

测试按钮位于**首页顶部右侧**，具体位置如下：

### 视觉位置

```
┌─────────────────────────────────────────┐
│  CDC 智汇中心              [🧪] [➕]   │  ← 这里！
│  学院信息智能聚合平台                    │
│  [🔍 搜索框...]                         │
└─────────────────────────────────────────┘
```

### 按钮说明

- **🧪 蓝色按钮**（左侧）- 测试 Supabase 连接
- **➕ 紫色按钮**（右侧）- AI 识别功能（原有功能）

---

## 🎯 如果看不到按钮

如果页面显示空白或 "Hello World"，请按以下步骤操作：

### 步骤 1：刷新编译

1. 在微信开发者工具中
2. 点击顶部工具栏的 **"编译"** 按钮（或按 `Cmd/Ctrl + R`）
3. 等待编译完成

### 步骤 2：检查编译状态

查看底部面板的 **"构建"** 标签：
- ✅ 显示 "Compiled successfully" - 编译成功
- ❌ 显示错误信息 - 需要修复错误

### 步骤 3：清除缓存

如果还是看不到：

1. 在开发者工具中
2. 点击 **"工具"** → **"清除缓存"** → **"清除全部"**
3. 重新编译

---

## 🧪 测试步骤

### 1. 找到测试按钮

在首页顶部右侧，找到 🧪 蓝色按钮

### 2. 点击测试按钮

点击 🧪 按钮，会触发 Supabase 连接测试

### 3. 观察按钮状态

按钮颜色会变化：
- 🧪 **蓝色** - 未测试/准备测试
- ⏳ **灰色** - 测试中（请等待）
- ✅ **绿色** - 连接成功
- ❌ **红色** - 连接失败

### 4. 查看控制台

1. 点击 **"调试器"** → **"Console"** 标签
2. 查看输出：

**成功示例：**
```
🧪 开始测试 Supabase 连接...
✅ Supabase 连接成功！ [...]
📊 获取到 3 条数据
```

**失败示例：**
```
❌ Supabase 连接失败：Error: request:fail url not in domain list
```

---

## 🔧 如果页面是空白的

如果页面显示空白，可能是运行时错误。请检查：

### 1. 查看控制台错误

在 **"调试器"** → **"Console"** 中查看是否有红色错误信息

### 2. 常见错误

- **URL 构造错误** - 已修复 ✅
- **AbortController 错误** - 已修复 ✅
- **域名未配置** - 需要在微信公众平台配置

### 3. 检查 Network 面板

1. 点击 **"调试器"** → **"Network"** 标签
2. 刷新页面
3. 查看是否有失败的请求

## 💡 快速定位按钮的方法

如果找不到按钮，可以：

1. **查看页面结构**
   - 在开发者工具中，点击 **"调试器"** → **"Wxml"** 标签
   - 搜索 "🧪" 或 "testSupabaseConnection"
   - 找到对应的按钮元素

2. **使用控制台测试**
   ```javascript
   // 在控制台直接测试
   import { getEvents } from '../../utils/supabase'
   getEvents().then(events => {
     console.log('✅ 连接成功！', events)
   }).catch(error => {
     console.error('❌ 连接失败：', error)
   })
   ```
